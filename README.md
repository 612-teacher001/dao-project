# Javaマスター DAO導入研修について（運用ルール版）

## 🧭 研修用レポジトリについて

このレポジトリは Java
マスター第11章（DAOパターン導入・カートシステム）研修用です。

本リポジトリには、完成形のサンプル実装（参照用コード）を含みます。\
これは「正解コード」ではなく、最終到達イメージを確認するための参照実装です。

------------------------------------------------------------------------

## 💡 研修における基本方針

研修では、Git操作と確認方法を以下の3つに分離します。

-   👀 見る（仕様・完成形の確認） → ブラウザ（GitHub等）
-   ▶ 動かす（実行確認） → ローカルでブランチ切替
-   💾 残す（作業保存） → commit

------------------------------------------------------------------------

## 🧑‍💻 完成サンプルの扱い

完成サンプルは参照専用です。  
直接編集するブランチではありません。

### ■ 完成サンプルの確認

``` bash
git switch goal/cart-system
```

このブランチは以下の用途です：

-   完成形の動作確認
-   実装イメージの把握
-   参考コードの閲覧

※修正やコミットは行わないこと

------------------------------------------------------------------------

## 🔁 作業ブランチへの戻り方

作業中のブランチへ戻る場合：

``` bash
git switch -
```

または明示的に：

``` bash
git switch work
```

------------------------------------------------------------------------

## 📘 課題に取り組むフロー（共通）

研修中は以下の流れを基本とします。

1.  **作業用ブランチを作る**

    ``` bash
    git switch work
    ```
    
    → プロンプトの行末が「（work）」に表示が変わっていれば成功

2.  **完成サンプルを確認する**

    ``` bash
    git switch goal/cart-system
    ```

    → 仕様と動作イメージを把握する

3.  **作業ブランチに戻る**

    ``` bash
    git switch -
    ```

3.  **課題実装を行う**

    -   featureブランチ上で実装する

4.  **動作確認をする**（⚠️地味に重要）

5.  **作業をcommitする**

    ``` bash
    git add .
    git commit -m "課題対応"
    ```

※pushは指示がある場合のみ、以下の操作を行う

6.  **mainブランチに切り替えてマージする**

    ``` bash
    git switch main
    git merge work
    ```

7.  **mainをプッシュする**

    ``` bash
    git push origin main
    ```

8.  **workブランチに切り替えておく**

    ``` bash
    git switch work
    ```
------------------------------------------------------------------------

## 🧠 Git操作の意味

  | 操作 | 意味 |
  |------|------|
  | git switch | 作業場所（ブランチ）を切り替える
  | commit     | 作業の区切りを保存する
  | push       | 他人と共有する
  | git status | 現在の状態を確認する

------------------------------------------------------------------------

## ⚠️ 重要ルール

-   未commitの状態でブランチを切り替えると制限される場合がある
-   完成サンプル（goal/cart-system）は参照専用として扱う
-   「見たいだけ」はブラウザ確認を優先する

------------------------------------------------------------------------

## 🎯 判断ルール（迷ったらこれ）

-   仕様を見たい → ブラウザ
-   動きを確認したい → git switch
-   作業を残したい → commit
-   共有したい → push
